## 1. Business purpose
옛날부터 1인 가구, 특히 대학생 자취생들에게 ‘살림’은 늘 뒷전이기 일쑤였다.
학업과 아르바이트에 치이다 보면 냉장고 구석에 박혀 형체를 알 수 없게 변해버린 식재료를 발견하거나, 샤워를 하려다 뒤늦게 떨어진 샴푸를 확인하고 당황하는 일은 자취생들의 흔한 일상이자 고질적인 불쾌함이다.
단순히 귀찮음을 넘어, 유통기한이 지나 버려지는 음식물 쓰레기와 이미 집에 있는 생필품을 또 구매하게 되는 ‘중복 지출’은 자취생들의 얇은 지갑을 위협하는 실질적인 경제적 손실로 이어진다.
그러나 이러한 불편함에도 불구하고 기존의 해결책은 포스트잇에 수동으로 적어두거나 막연한 기억력에 의존하는 수준에 머물러 있다.
시중에 나온 대형 가전사의 스마트 냉장고는 자취생들이 구매하기엔 지나치게 고가이며, 일반적인 메모 앱은 유통기한 알림이나 소비 패턴 분석 기능을 제공하지 않아 체계적인 관리가 불가능하다.
그래서 이러한 자원 낭비를 막고, 자취생들이 좁은 주거 공간에서도 쾌적하고 경제적인 생활을 누리도록 돕고 싶었다. 그리하여 데이터 기반의 자취생 맞춤형 재고 관리 시스템인 **“HIMS: Household Inventory Management System”**를 개발하게 되었다.
이 프로그램을 통해 이루고자 하는 첫 번째 목표는 철저한 자산 및 재고의 최적화다. 사용자별 독립된 데이터베이스를 구축하여 내가 가진 모든 생필품의 수량과 유통기한을 추적한다.
'딸깍' 한 번의 입력으로 재고 현황을 디지털화하고, 유통기한 임박 시 시각적 알림을 제공하여 버려지는 자원을 원천 차단하는 것이 첫 번째 목표이다.
그 다음으로는 데이터 기반의 현명한 소비 가이드다. 단순히 쌓아두는 것을 넘어, 사용자의 누적된 소비 데이터를 분석하여 품목별 교체 주기와 월간 소비량을 통계로 보여준다.
이를 통해 본인의 소비 습관을 파악하고 불필요한 과잉 소비를 줄이는 것이 두 번째 목표이다.
본 시스템은 단순히 개인의 관리에만 국한되지 않는다. 시스템 관리자(Admin)가 주변 마트의 할인 정보나 공동구매 소식을 실시간으로 배포함으로써, 사용자는 더 저렴하게 물품을 보충하고 관리자는 유효한 정보를 전달하는 **'자취생 통합 정보 허브'**로 그 범위를 무한히 확장할 수 있다.
이 시스템의 주 타겟은 매달 한정된 생활비 안에서 효율적인 살림 운영이 절실한 모든 1인 가구 및 대학생 자취생이 될 것이다.
학교 인근 원룸촌이나 오피스텔에 거주하며 식재료 관리에 어려움을 겪는 학생들은 물론, 첫 독립을 시작하여 살림 노하우가 부족한 사회 초년생들에게 필수적인 생활 도구로 자리 잡을 수 있다.
결과적으로 **“HIMS”**는 단순히 물건 개수를 세는 장치를 넘어, 1인 가구의 경제적 자립과 주거 환경의 쾌적화라는 두 마리 토끼를 잡는 시스템이다. 이를 통해 자취 생활의 번거로움을 '스마트한 관리'와 '절약의 즐거움'으로 바꾸어 주는 차세대 생활 밀착형 솔루션을 지향한다.

## 5. Problem statement
‘HIMS(Household Inventory Management System)’는 1인 가구가 주거 공간 내 생필품 및 식재료의 재고 현황을 실시간으로 파악하고, 관리자가 제공하는 지역 할인 정보를 통해 효율적인 소비를 할 수 있도록 돕는 것이 목적이다. 
이 시스템은 다음과 같은 기술적/운영적 목적을 달성해야 한다.
- Firebase를 활용한 실시간 재고 수량 동기화
- 일반 사용자(User)와 시스템 관리자(Admin)의 명확한 권한 분리
- 사용자 소비 데이터를 활용한 통계 분석 모듈 구축

#### 1) Problem #1: 데이터 무결성 및 동기화 오류
재고 관리 시스템에서 가장 치명적인 문제는 실제 수량과 시스템상의 수량이 일치하지 않는 것이다. 네트워크 불안정 등으로 인해 전산 누락이 발생할 경우, 사용자는 잘못된 재고 정보를 바탕으로 불필요한 중복 구매를 하게 되어 시스템의 신뢰성을 상실하게 된다. 따라서 실시간 동기화 오류를 최소화하는 설계가 필수적이다.

#### 2) Problem #2: 권한에 따른 기능 접근 제어
HIMS는 관리자와 사용자의 역할이 엄격히 구분된다. 관리자만이 할인 정보를 게시하고 카테고리를 편집할 수 있어야 하며, 일반 사용자가 관리자 전용 DB 섹션에 접근하거나 정보를 수정하는 행위는 시스템 보안상 절대적으로 차단되어야 한다.

#### 3) Problem #3: 개인 정보 보호 및 보안 리스크
사용자의 소비 패턴과 구매 이력은 민감한 개인 정보에 해당한다. Firebase를 사용함에 있어 적절한 보안 규칙을 설정하지 않으면 외부 해킹이나 데이터 유출 사고로 이어질 수 있다. 따라서 사용자 UID를 기반으로 본인의 데이터에만 접근할 수 있는 철저한 보안 설계가 요구된다.

#### 4) Problem #4: 데이터 입력의 지속성 및 사용자 경험(UX)
시스템의 기능이 아무리 뛰어나도 사용자가 매번 제품명과 카테고리를 직접 타이핑하는 과정이 번거롭다면 사용자는 금방 이탈할 것이다. 입력 과정을 간소화할 수 있는 자주 쓰는 품목(Preset) 기능이나 카테고리 기반 자동 완성 시스템이 부족할 경우, 시스템의 실용성이 낮아질 위험이 있다.

#### 5) Problem #6: 분석 알고리즘의 정확도
사용자에게 제공되는 소비 패턴 분석 통계가 실제 소비 습관을 제대로 반영하지 못한다면 서비스의 가치가 떨어진다. 단순 누계가 아닌, 품목별 교체 주기와 소비 속도를 정확히 계산해낼 수 있는 로직 구현에 많은 시간을 할애해야 한다.

본 프로젝트는 HIMS 시스템의 신뢰성과 효율성을 보장하기 위해 다음과 같은 비기능적 요구 조건을 달성해야 한다.

1. **실시간 데이터 동기화 성능**
- 사용자가 재고 수량을 변경했을 때, Firebase Cloud DB에 반영되고 앱 화면에 업데이트되는 지연 시간은 네트워크 정상 환경 기준 1.5초 미만이어야 한다. 

2. **시스템 보안 및 접근 제어**
- Firebase Security Rules를 통해 사용자 UID 기반의 데이터 격리를 구현하며, 허가되지 않은 사용자의 Admin 데이터 섹션 접근 시도 시 100% 차단되어야 한다.

3. **데이터 처리 및 분석 속도**
- 소비 패턴 분석 모듈이 대량의 누적 데이터를 처리하여 통계 그래프를 생성하는 시간은 3초 이내여야 하며, 분석 오차 범위는 실제 입력 데이터 대비 1% 미만을 유지해야 한다. 

4. **입력 편의성 및 가용성**
사용자가 자주 구매하는 10개 이상의 품목을 **'즐겨찾기(Preset)'**로 등록하여 1회 클릭(One-tap)만으로 재고 등록이 가능해야 하며, 시스템은 연중무휴 99% 이상의 가동률을 보장해야 한다.

5. **네트워크 예외 처리**
- 오프라인 상태에서 발생한 재고 변경 사항은 로컬 캐시에 즉시 저장되어야 하며, 네트워크가 재연결되는 즉시 누락 없이 DB와 최종 동기화되어 데이터 무결성을 유지해야 한다.

## 6. Glossary

| Term | Description |
| :--- | :--- |
| **HIMS** | Household Inventory Management System의 약자로, 본 프로젝트에서 개발하는 가계 재고 관리 시스템의 명칭이다. |
| **일반 사용자 (User)** | 본인의 생필품 및 식재료 재고를 등록하고, 소비 패턴 분석 및 할인 정보를 조회하는 주체이다. |
| **관리자 (Admin)** | 시스템의 표준 카테고리를 관리하고, 지역 마트의 할인 정보를 전체 사용자에게 배포하는 운영 주체이다. |
| **인벤토리 (Inventory)** | 가정 내에 보유 중인 생필품, 식재료 등의 모든 재고 물품을 의미한다. |
| **재고 현황 (Stock Status)** | 각 품목별 현재 수량, 유통기한, 부족 여부 등을 실시간으로 나타내는 상태 데이터이다. |
| **즐겨찾기 (Preset)** | 사용자가 자주 구매하는 품목을 미리 등록하여, 별도의 입력 없이 '딸깍' 한 번으로 재고를 추가할 수 있게 돕는 기능이다. |
| **소비 패턴 분석** | 사용자의 입출력 데이터를 바탕으로 품목별 소진 속도 및 월간 소비량을 수치화하여 시각적으로 보여주는 기능이다. |
| **재고 알림 (Stock Alert)** | 설정된 최소 유지 수량보다 현재 재고가 부족해질 경우, 시스템이 사용자에게 구매 시점을 알리는 기능이다. |
| **데이터 무결성** | 실제 집 안의 재고 수량과 시스템 DB에 저장된 수치가 누락이나 오류 없이 일치하는 상태를 의미한다. |
| **할인 큐레이션** | 관리자가 검증한 지역 내 마트의 세일 정보나 유통기한 임박 할인 정보를 사용자에게 맞춤형으로 제공하는 서비스이다. |
| **카테고리 (Category)** | 식품, 세제, 위생용품 등 품목의 성격에 따라 데이터를 분류하는 표준 체계이다. |
